## Possible assessments

- Documentation and availability sufficient for the declared purpose
- Installation, tests or core functions under a named environment
- Usability of a specified dataset or synthetic companion package
- Execution of a bounded workflow and comparison of named outputs
- Synthetic-data utility or privacy under specialist criteria
- Authorised verification against controlled real data

## Required report

Every finding identifies object and version, evaluator and conflicts, date, criteria, environment, commands or procedures, evidence, tolerances, successful outcomes, differences, failures, excluded components and limitations. Partial or unsuccessful checks remain visible and informative.

## Critical distinctions

Functional software is not the same as reproduced research results. Reproduction of results is not independent replication. A synthetic workflow run is not secure real-data verification. A technical evaluator does not make the journal’s scientific decision.

Qualifying evidence may support an [evidence badge](../badges/), but badge approval requires human review and publication acceptance.
