A fictional medical-AI Open Research Record.

This worked example shows how JORN could connect controlled patient data, synthetic companion data, open software, external review and bounded verification claims.

Illustrative example—not a real JORN publication. Every person, institution, identifier, DOI, community, assessment, date, result and quotation on this page is fictional. Nothing below records an actual review, partnership, clinical result, certification or publication.

Scholarly abstract — fictional¶

This fictional study describes a model intended to prioritise retinal images for specialist review. The fictional authors report an evaluation against controlled patient images held in a fictional secure environment. They publish fictional open-source software and an openly licensed synthetic companion dataset for demonstrating the workflow. The example is designed only to show JORN’s record structure; it makes no real scientific or clinical claim.

Plain-language summary — fictional¶

A fictional research team tested fictional software on fictional protected clinical images. Readers could use a made-up synthetic dataset to see how the workflow is structured, but that dataset would not reproduce the clinical evaluation. A fictional authorised evaluator separately checked selected results inside a fictional secure environment.

Editorial summary — fictional¶

Review route: coordinated external-community route involving the Fictional Clinical Imaging Review Collective [FICTIONAL COMMUNITY; NOT A JORN PARTNER]. The fictional community coordinated a clinician and statistical reviewer; a fictional JORN handling editor appointed a privacy reviewer and artifact evaluator. The fictional decision cited clear versioning and bounded secure verification while noting limited demographic and external-site evidence. All strengths, limitations and outcomes in this section are invented.

AI-assisted triage statement — fictional¶

Fictional JORN Triage System 0.4 analysed only the fictional public preprint, metadata, synthetic dataset and public software on 2 February 2028 against fictional criteria 0.3. It did not receive patient data or confidential reviews. It identified a missing software licence and unclear subgroup documentation; the fictional authors corrected both. A fictional editor confirmed “ready for human review.” AI suggested expertise but neither selected reviewers nor made the publication decision. No such system or screening occurred.

Data and synthetic-data statement — fictional¶

The fictional patient data remain controlled because the fictional consent and governance terms do not permit public release. A public fictional metadata record describes the cohort, variables and access process. The synthetic companion dataset was generated with fictional tool SynthExample 1.2 and supports input validation and workflow demonstration. A fictional disclosure-risk assessment tested membership and attribute-inference scenarios. It did not establish anonymity, clinical fidelity, or result reproduction.

The synthetic dataset underwent the disclosure-risk assessment described in this record. [FICTIONAL CLAIM]

Software and external assessment — fictional¶

The fictional development repository links to archived release 1.0, environment 1.0, tests, documentation and CITATION.cff. Fictional Software Review Collective assessed release 1.0 under fictional criteria 2.1. JORN fictionally recognised only documentation, testing and maintainability findings—not scientific validity or clinical safety. The organisation, review and recognition do not exist.

Scientific review record — fictional¶

  • Fictional clinician report: scope, intended use, clinical interpretation and limitations
  • Fictional statistical-methods report: design, discrimination and calibration analyses
  • Fictional privacy report: governance, access and disclosure-risk claims
  • Fictional author response and versions v1–v3
  • Fictional handling editor decision dated 15 May 2028

Public comments were not treated as commissioned review. Reviewer identities remained fictional and confidential. No real review was performed.

Artifact and secure verification — fictional¶

A fictional evaluator ran release 1.0 using environment 1.0 and the synthetic companion data. A separately authorised fictional evaluator ran specified evaluation scripts inside the fictional secure archive against controlled dataset v2. Checksums and aggregate comparison outputs were exported under fictional governance approval. The reported primary aggregate metrics matched the fictional manuscript within a predeclared fictional tolerance. No patient records left the environment.

Certification and badges — fictional¶

JORN Certified Open Research Record — FICTIONAL. This example pretends that JORN peer-reviewed, accepted, published and connected the exact fictional versions above. JORN has issued no such certification.

  • Artifacts Evaluated—Functional — FICTIONAL: specified software and documentation functioned for the declared test workflow.
  • Computational Results Reproduced — FICTIONAL: specified aggregate outputs were obtained within fictional tolerances.
  • External Software Review Recognised — FICTIONAL: the scope of a fictional software review was recognised.
  • Secure Real-Data Verification — FUTURE, FICTIONAL CLAIM: specified scripts and aggregate results were checked inside a fictional secure environment.

What fictional JORN verified¶

In this scenario only: exact versions and relationships; presence and documented access route for controlled data; synthetic workflow function; bounded aggregate secure comparison; qualifying external software-review scope; review and revision history.

What was not assessed — fictional limitations¶

No independent replication, prospective clinical evaluation, deployment safety assessment, regulatory approval, generalisation beyond the fictional sites, complete source-data release, guarantee of privacy, or guarantee of correctness. Rare-condition and minoritised-subgroup evidence remained limited. The synthetic data did not reproduce clinical findings.

Ethics, privacy and governance — fictional¶

Fictional Ethics Committee approval EX-000 and fictional controlled-access agreement CA-000 are placeholders. JORN never received patient-level data. All governance bodies, approvals and procedures described here are invented and must not be relied upon.

How to cite¶

Do not cite this as research or a JORN publication. If discussing the website design, cite it as: “JORN, Illustrative Open Research Record, website example, accessed [date], explicitly fictional and not a publication.” The fictional DOI, BibTeX, RIS, CSL JSON and EndNote XML are intentionally unavailable.
